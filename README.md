# Bloq

Браузерное расширение (Chrome / Edge / любой Chromium, Manifest V3):

1. **Реклама** - сетевые запросы к рекламным сетям и трекерам режутся через
   `declarativeNetRequest` ещё до загрузки (`rules/ads.json`).
2. **Баннеры** - косметическое скрытие рекламных контейнеров CSS-правилами
   (`data/selectors.mjs` -> `cosmeticSelectors`).
3. **Cookie-попапы** - окна согласия скрываются, в 11 известных CMP
   автоматически нажимается «Отклонить все» (ставится отказный consent-cookie,
   и баннер не возвращается), блокировка скролла страницы снимается.

## Установка

1. Открой `chrome://extensions`.
2. Включи «Режим разработчика» (справа сверху).
3. «Загрузить распакованное расширение» -> укажи эту папку.

На иконке появится бейдж со счётчиком заблокированных запросов на вкладке.

## Использование

- Клик по иконке - попап:
  - **Блокировка включена** - глобальный тумблер;
  - **Блокировать на [сайт]** - выключи, чтобы внести сайт в allowlist
    (сетевые правила применяются после обновления страницы, косметика - сразу).
- Allowlist работает по домену с поддоменами; ведущий `www.` отбрасывается.

## Устройство

```
manifest.json            MV3-манифест
rules/ads.json           статические DNR-правила (сетевая блокировка)
data/selectors.mjs       селекторы косметики и cookie-CMP (общие с тестами)
common/matching.mjs      сопоставление хостов (background + popup + content + тесты)
src/background.js        service worker: состояние, allowlist-правила, бейдж
src/content/bloq.js      контент-скрипт: скрытие, авто-отказ, скролл
src/popup/               попап
tools/make-icons.mjs     генератор иконок (npm run icons)
tests/                   node --test, без зависимостей
```

Принятые решения:

- **Block-правила бьют только third-party и никогда main_frame** - иначе
  ломаются рекламные кабинеты (ads.google.com и т.п.) и прямые заходы на сайты
  сетей. Это закреплено тестом (`policyErrors`).
- **CMP-скрипты сетево не блокируются** - часть сайтов ждёт callback от CMP и
  ломается без него. Вместо этого попап скрывается косметикой + авто-отказ.
- **Один источник данных** - селекторы и матчинг хостов лежат в `.mjs`-модулях,
  которые импортируют и расширение (динамический `import` в контент-скрипте,
  статический в background/popup), и тесты. Правишь список - меняется и
  поведение, и то, что проверяют тесты.

## Как добавить

- **Домен в блок**: допиши в подходящее правило `rules/ads.json`
  (`condition.requestDomains`). Домен матчится вместе с поддоменами.
- **Баннер**: селектор в `cosmeticSelectors` (`data/selectors.mjs`).
- **Cookie-попап**: селектор контейнера в `cookieHideSelectors`; если знаешь
  кнопку «Отклонить все» - добавь запись в `cookieRejectClicks`.

После правок прогони тесты.

## Тесты и сборка

```
npm test        # node --test - проверки без зависимостей
npm run icons   # перегенерировать icons/*.png
npm run build   # собрать dist/bloq-<версия>.zip для Chrome Web Store
```

Публикация в Chrome Web Store: пошагово - [PUBLISHING.md](PUBLISHING.md), готовые тексты
карточки - [STORE_LISTING.md](STORE_LISTING.md), политика - [PRIVACY.md](PRIVACY.md),
ассеты витрины - `store-assets/`.

Без зависимостей (Node 18+). Что проверяется: схема и политика DNR-правил,
синтаксическая безопасность всех селекторов, границы матчинга доменов
(включая суффикс-атаку `evil-example.com`), целостность манифеста (все файлы
на месте, web_accessible_resources покрывают динамические import),
единство версии в manifest/package/CHANGELOG, валидность PNG-иконок.
Негативные тесты кормят валидаторам заведомый брак.

## Ограничения

- Списки курируемые и компактные, а не EasyList на 100k правил: самые
  агрессивные сети и CMP покрыты, но экзотика может проскочить.
- Нет механизма per-site исключений для косметики (только allowlist всего
  сайта) - если генерик-селектор скрыл что-то нужное, отключи Bloq на
  сайте через попап.
- Firefox не поддержан (другая модель фоновых скриптов MV3).
